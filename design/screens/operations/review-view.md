## 1. Design Intent

Reviews are the enforcement mechanism for the SOP engine. Without regular reviews, tasks pile up, projects drift, and objectives fail silently. The Review View makes reviews effortless by pre-generating all the data and presenting it as a guided checklist.

The heartbeat process triggers reviews at the right cadence. When triggered, an input_queue item appears. Opening it launches the Review View with all relevant data pre-loaded.

## 2. Review Types & Templates

### 2.1 Daily Review (auto-generated at evening close)

**Sections:**

1. **Today's Completions** â€” Tasks marked done today. Count + list. XP earned.
2. **Carry-overs** â€” Tasks scheduled today but not completed. For each: [Reschedule to tomorrow] [Reschedule to...] [Drop]
3. **Input Queue Status** â€” Items resolved today / items still pending.
4. **Blockers** â€” Any tasks or projects that became blocked today.
5. **Tomorrow Preview** â€” Pre-scheduled tasks for tomorrow. [Looks good] or [Adjust]
6. **Quick Reflection** â€” Optional text field: "What went well? What didn't?"

**Auto-generated by:** Heartbeat evening close (20:00).
**Duration target:** 2-3 minutes.

### 2.2 Weekly Review (Friday)

**Sections:**

1. **Week Summary** â€” Tasks completed (count), tasks created, input queue throughput, XP earned, cost spent.
2. **Project Status Check** â€” Each active project with: status, progress delta this week, tasks completed/remaining, owner. For each: [On Track] [At Risk] [Blocked] [Pause] [Complete]
3. **Stale Input Queue** â€” Items pending > 3 days. For each: [Resolve now] [Dismiss] [Snooze]
4. **Dependency Audit** â€” Bottleneck report: tasks blocking the most other tasks. Highlight critical path.
5. **Principle Capture** â€” "Did you make any decisions this week that should become principles?" [Add principle] or [Skip]
6. **Next Week Planning** â€” Pre-populated tasks for next week. Drag to reorder/reschedule. Assign time blocks.
7. **Agent Performance** â€” Approval rates, redo rates, cost per agent this week.

**Duration target:** 15-20 minutes.

### 2.3 Monthly Review (Last day of month)

**Sections:**

1. **Objective Progress** â€” Each active objective with key result progress bars. Delta since last month.
2. **Project Health** â€” Projects that are behind schedule. Decision: [Accelerate] [Rescope] [Pause] [Kill]
3. **Agent Review** â€” Monthly stats per agent. Trends over time.
4. **Cost Summary** â€” Total agent cost, cost per area, cost per project.
5. **Insights** â€” Agent-generated summary of patterns (most productive days, common blockers, etc.)

**Duration target:** 30 minutes.
